# Support
## Where to ask for help
- Use repository issues for actionable bugs and concrete feature requests.
- Use repository discussions when a repository has discussions enabled and the question is exploratory.
- Use pull requests for proposed fixes with implementation context.

## Security issues
Do not use public issues for vulnerabilities or sensitive exposures. Follow `SECURITY.md`.

## Good support requests
Include the repository, branch or revision, expected behavior, actual behavior, and the smallest reproduction you can provide.
